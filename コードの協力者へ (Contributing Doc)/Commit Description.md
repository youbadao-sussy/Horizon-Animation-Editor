# 注意 (English is posted below)
協力してくれる方への注意点を書いておきます。
コミットする際は、コミットメッセージを`<type>: <description>`という形式にしてください。

## 書き方
`<type>`には、一つの変更につき以下の文字のどれか一つを記入してください。

* `doc/Doc`: 長い説明や注意点を書くとき (機能の削除など)
* `feat/Feat`: 新たな機能を追加したとき
* `fix/Fix`: バグの修正
* `refactor/Refactor`: コードの簡略化や整理、改善
* `perf/Perf`: パフォーマンスの最適化
* `style/Style`: UIの変更
* `test/Test`: ...まだ悩み中
* `ci`: ...まだ悩み中
* `security/Security`: セキュリティ関連の修正、追加(GitHubのフォルダ共有者のみ)
* `revert/Revert`: 削除または変更された機能を戻す(GitHubのフォルダ共有者のみ)
* `gitedit/Gitedit`: GitHub系ファイルの追加、変更、修正

`<description>`には変更点などの説明を記入してください。

# Notes for Contributors
Please note the following points for contributors.
When committing, format your commit message as `<type>: <description>`.

## Writing Format
For `<type>`, use one of the following characters per change:

* `doc/Doc`: For lengthy explanations or notes (e.g., feature removals)
* `feat/Feat`: When adding a new feature
* `fix/Fix`: Bug fixes
* `refactor/Refactor`: Code simplification, organization, or improvement
* `perf/Perf`: Performance optimization
* `style/Style`: UI changes
* `test/Test`: ...still considering
* `ci`: ...still considering
* `security/Security`: Security-related fixes or additions (GitHub folder sharers only)
* `revert/Revert`: Restoring removed or changed features (GitHub folder sharers only)
* `gitedit/Gitedit`: Adding, modifying, or fixing GitHub-related files

Please enter a description of the changes in `<description>`.
